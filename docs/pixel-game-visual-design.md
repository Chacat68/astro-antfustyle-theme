# 像素游戏视觉方案：《铁头 · Arcana》

> 面向 Godot 4.x 的 **现代像素风平台跳跃**。关卡短平快（60–120 秒）、可程序化重组；视觉目标是：读得清、跳得爽、死得明白、过关有仪式感。  
> 本文是 UI / 画面的落地规范，后续实现与迭代以本文为准，改动后请同步回写。

---

## 1. 视觉总纲

### 1.1 一句话气质

**「盔甲骑士在重组中的魔法王国里闯关」**——厚重但不沉闷，像素但清晰，魔法感来自光色与粒子，不靠紫雾滤镜堆氛围。

### 1.2 风格锚点

| 维度 | 约定 |
|------|------|
| 分辨率基准 | 逻辑分辨率 **320×180**（16:9），整数倍放大到窗口（×3 / ×4 / ×5） |
| 像素密度 | 角色约 **16×16～24×24**；瓦片 **16×16**；UI 控件按 8px 网格对齐 |
| 色深 | 限制调色板（约 32～48 色全局），允许局部叠色与 1px 高光 |
| 透视 | 严格侧视 2D；背景用 **2～3 层视差**，不用伪 3D |
| 动画 | 角色 8～12 FPS 关键帧 + 60 FPS 位移；UI 用帧动画或阶梯式缓动，禁止平滑矢量插值糊边 |
| 缩放 | 最近邻（Nearest）；禁止双线性模糊像素 |

### 1.3 色彩系统

以 **铸铁灰 + 炼金琥珀 + 夜空靛** 为主，避免默认「紫白渐变」「奶油陶土」模板感。

```
--bg-deep          #0E1218   深夜底（菜单/过场）
--bg-play          #1A2330   关卡天空基准（可按章节换 tint）
--surface          #243044   面板底
--surface-raised   #2E3D55   抬起按钮/选中条
--border           #4A5D78   1px 像素描边
--text-primary     #E8EEF6   主文案
--text-muted       #8A9BB0   次要说明
--accent-amber     #E8A838   主 CTA / 金币 / 过关高光
--accent-steel     #7EB6D9   次要交互 / 链接 / 法术提示
--danger           #D64545   受伤 / 危险 / 死亡
--safe             #4CAF7A   回复 / 成功 / 可站立提示
--arcana           #5BC4C4   Arcana 魔法（青色，非紫色）
--knight           #C9B08A   铁头盔甲受光色
```

**章节染色**：村庄偏暖琥珀、城堡偏冷钢蓝、地下偏靛青；只改背景 tint 与远景层，不重做整套 UI 色板。

### 1.4 字体

| 用途 | 方案 | 说明 |
|------|------|------|
| 标题 / Logo | 像素显示字体（如 `m5x7` / 自制 8×8 位图字） | 字号 16 / 24，字距固定 |
| 正文 / 菜单 | 同一像素字体 8～10px | 中文用专用像素中文字模，或位图字图集 |
| 数字 HUD | 等宽像素数字（8×8） | 分数、计时、生命必须等宽防抖动 |

禁止在游戏内使用 Inter / 系统无衬线作为主 UI 字体。若调试构建需要，可临时 fallback，发布包必须切回像素字。

### 1.5 构图与信息层级

1. **一屏一事**：标题、设置、结算各自只承担一个任务。  
2. **玩法优先**：局内 HUD 永远让路给角色与落脚点。  
3. **危险可读**：尖刺、落坑、弹幕用高对比轮廓 + 固定危险色，不靠特效掩盖。  
4. **反馈即语言**：跳跃落地、受伤闪白、踩敌弹跳都用短粒子 + 屏幕微震，而不是大段文字。

### 1.6 动效原则（整机共用）

- 菜单焦点：选中项左右 **2px 像素跳动** + 琥珀色短下划线闪烁（4 帧循环）。  
- 面板出现：自下向上 **8px 阶梯滑入**（3～4 步），带 1 帧遮罩。  
- 尊重 `prefers-reduced-motion` / 无障碍选项：可关闭屏幕震动与闪白。  
- 单次反馈时长：UI 150～250ms；战斗受击 80～120ms；过关仪式 1.2～2.0s。

## 3. 各页面详细设计

### 3.1 启动闪屏（Boot / Splash）

**目的**：品牌露出 + 资源预热，不抢戏。

| 项 | 规范 |
|----|------|
| 时长 | 1.5～2.5s，可按任意键跳过（首次启动除外可强制看完 Logo） |
| 背景 | `--bg-deep` 纯色，无复杂动画 |
| 中心 | 工作室 / 项目像素 Logo，最大宽约 120px |
| 底部 | 一行小字「Press any key」，琥珀色 2Hz 闪烁 |
| 转场 | 硬切或 4 帧像素溶解到标题页 |

**不要做**：进度条百分比大字、多 Logo 轮播、过长动画。

---

### 3.2 标题主菜单（Title）

**目的**：建立世界观第一印象，并给出唯一明确行动「开始」。

#### 构图（320×180）

```
┌────────────────────────────────────────┐
│  远景：重组中的城堡剪影（视差慢漂）        │
│                                        │
│     铁头立绘（右下 1/3，待机呼吸帧）       │
│                                        │
│  ┌ 标题区（左上偏中）                    │
│  │  铁头 · ARCANA                      │
│  │  一句副文案（≤16 字）                 │
│  └                                     │
│                                        │
│  ┌ 菜单（左下）                          │
│  │  ▶ 开始冒险                          │
│  │    继续游戏                          │
│  │    设置                              │
│  │    退出                              │
│  └                                     │
│  版本号 v0.x.x（右下，muted）            │
└────────────────────────────────────────┘
```

#### 设计要点

- **品牌优先**：标题字标是第一视觉，副文案不得比标题更大。  
- **一张主视觉**：全屏视差背景即主画面，不做卡片墙、不做悬浮徽章。  
- **菜单**：纯文字列表 + 像素箭头焦点，不要按钮卡片。  
- **铁头**：半身或全身待机（盔甲微亮、面甲开合 1～2 帧），暗示「情绪反馈」设定。  
- **Arcana 痕迹**：远处建筑轮廓偶发青色扫描线（极淡），暗示地图会重组。

#### 交互

- 上/下切换；确认进入；取消无操作或打开退出确认。  
- 「继续」仅在有存档时显示；无存档时该项隐藏，不置灰占位。  
- 首次进入可播 0.5s 标题字标「敲击落地」帧动画（盔甲落地尘土粒子 6～8 颗）。

### 3.3 设置页（Settings）

**目的**：调音量、画面、操作、无障碍；一屏完成，可分页但不嵌套过深。

#### 布局

```
┌─ 设置 ──────────────────────── ╳ 关闭 ─┐
│  [游戏] [画面] [音频] [操作] [无障碍]     │  ← 顶部分类 Tab（像素下划线）
│                                        │
│  主音量        ████████░░  80%          │
│  音效          ██████████ 100%          │
│  屏幕震动      [开]                     │
│  闪白受击      [开]                     │
│  窗口模式      全屏 ▾                   │
│  整数缩放      ×4 ▾                    │
│  语言          中文 ▾                   │
│                                        │
│  提示：方向键调节 · Enter 确认 · Esc 返回 │
└────────────────────────────────────────┘
```

#### 视觉

- 模态面板居中，宽约 220×140；半透明暗幕（`--bg-deep` @ 70%）。  
- 面板 1px `--border` 描边；标题条用 `--surface-raised`。  
- 滑条为像素块进度，不用圆形 thumb。  
- 开关为两位图：空框 / 琥珀实心勾。

#### 分类内容建议

| Tab | 内容 |
|-----|------|
| 游戏 | 语言、显示伤害数字、显示计时 |
| 画面 | 全屏、整数缩放、CRT 扫描线强度（默认关）、色盲辅助 |
| 音频 | 主/音效/音乐分轨 |
| 操作 | 键位重绑列表；提供「恢复默认」 |
| 无障碍 | 关闭震动、关闭闪白、增大 HUD、高对比危险色 |

从标题进入与从暂停进入共用同一场景，仅返回目标不同。

### 3.4 章节 / 种子选择（Run Setup）

**目的**：选难度或世界种子后开跑；保持「短平快」节奏，不做成大地图养成。

#### 方案 A（MVP 推荐）：简洁开跑页

```
┌─ 出发 ─────────────────────────────────┐
│  难度                                   │
│  ○ 见习   ● 骑士   ○ 铁壁               │
│                                        │
│  世界种子  [ AUTO ]  或  [自定义……]      │
│                                        │
│  本跑规则摘要（3 行内）：                 │
│  · 生命 ×3 · 踩踏攻击 · 关卡重组         │
│                                        │
│         [ 进入王国 ]                    │
└────────────────────────────────────────┘
```

- 难度用横向三选一，选中项琥珀描边 + 铁头小图标姿态变化（见习抬头 / 骑士持剑 / 铁壁蹲姿）。  
- 「进入王国」为唯一主 CTA，放在面板底部居中。

#### 方案 B（后续）：章节灯塔地图

横向卷轴式 **灯塔 / 城寨节点**（非卡片网格）：已通关亮琥珀，当前关呼吸光，未解锁为剪影。点击节点只显示一行描述 + 进入，不弹多层。

### 3.5 Loading

**目的**：掩盖关卡实例化；传达「Arcana 正在重组」。

| 项 | 规范 |
|----|------|
| 背景 | `--bg-deep` |
| 中心 | 青色六边形符文缓慢旋转（步进旋转，每 100ms 一格，非平滑） |
| 文案 | 「Arcana 重组中…」+ 一句随机短提示（手感 / 踩敌 / 种子） |
| 进度 | 可选细像素条；失败重试时改为「重新编织…」 |
| 时长目标 | 理想 < 800ms；超过 1.5s 才显示完整动画 |

禁止假进度条长期卡 99%。

---

### 3.6 局内 HUD（Gameplay）

**目的**：最少信息、最快读取；任何装饰不得遮挡跳跃判定视线。

#### 安全区与锚点（320×180）

```
生命 ●●●          计时 01:12          金币 012
（左上）           （顶中，可关）        （右上）

                 [ 角色活动区 ]

快捷：Esc 暂停                    章节小标（可选，淡）
```

#### 元素规范

| 元素 | 样式 | 规则 |
|------|------|------|
| 生命 | 3 个盔甲徽记（实心/空心） | 受伤时对应徽记碎裂 3 帧；回复拼合 |
| 计时 | 等宽数字 | 默认开；紧张关可变为琥珀闪烁（最后 10s） |
| 金币 / 收集 | 图标 + 等宽数 | 拾取时数字上跳 4px 并淡出 +1 |
| Boss / 特殊 | 底部细血条 | 仅特殊关出现；普通关不占位 |
| 教程提示 | 底部一行半透明字 | 3s 后淡出，可设置永久关闭 |

#### 反馈叠加（非常驻 UI）

- **受伤**：全屏 2～3 帧红色边缘闪 + 角色白闪；可关。  
- **踩敌成功**：短星形像素粒子 + 角色额外起跳弧线。  
- **落地尘土**：4～6 颗粒，按地面材质换色。  
- **靠近悬崖 / 尖刺**：不弹窗；危险物自身高亮轮廓即可。

#### 禁止

- 首屏堆统计条、任务列表、广告式角标。  
- 半透明大面板常驻。  
- 圆形大头像框、技能转盘（本作为平台跳跃，能力用解锁演出，不靠复杂 HUD）。

### 3.7 暂停菜单（Pause）

**目的**：立刻停下；提供继续、设置、放弃本跑。

```
┌──────── 暂停 ────────┐
│  ▶ 继续冒险           │
│    设置               │
│    重新开始本关       │
│    返回标题           │
└──────────────────────┘
```

- 背景：局内画面冻结 + 暗幕 50%；**不做模糊滤镜**（像素风模糊易脏）。  
- 面板小而居中，约 120×90。  
- 「返回标题」需二次确认小框：「结束本次冒险？」→ 确定 / 取消。  
- 打开暂停时切断输入到角色；关闭后给 3 帧输入缓冲，避免误操作。

---

### 3.8 对话 / 过场（Dialog & Cutscene）

**目的**：交代 Arcana / 梅林 / 铁头动机；可跳过。

#### 对话框

```
┌──────────────────────────────────────────┐
│ [头像 32×32]  梅林                         │
│  王国正在被「完美秩序」重写。你，还走得动吗？ │
│                                ▼          │
└──────────────────────────────────────────┘
```

- 位置：屏幕下方，高度约 48px，不挡角色脚点时可上移。  
- 头像：像素立绘，说话时 2 帧口型或盔甲微动。  
- 文本：逐字显示（每 2 帧一字）；按确认键可加速 / 跳过当前句。  
- 选项分支（若有）：最多 3 项，纵向排列在对话框上方，焦点同主菜单。

#### 过场

- 全屏黑底 + 中间插画条（180 宽内），或局内摄像机锁定演出。  
- 可跳过提示固定右下：「Hold 跳过」。  
- 重要能力解锁用 **盔甲部件装配动画**（护手/护胫/胸甲依次扣上），作为世界观与成长的视觉锚点。

---

### 3.9 死亡与续关（Death）

**目的**：快速重来；说明死因但不说教。

```
        铁头倒地剪影（短循环）

        盔甲裂了……
        死因：尖刺 / 坠落 / 敌人（一小行 muted）

        ▶ 再试一次
          返回标题
```

- 背景：局内最后一帧压暗至 30% 亮度，或切 `--bg-deep`。  
- 「再试一次」默认焦点，确认后 **同种子重开**，Loading 尽量跳过或极短。  
- 连续死亡 3 次可在角落淡出提示：「设置里可调节震动与显示计时」（不强制新手模式弹窗）。

---

### 3.10 过关结算（Level Clear）

**目的**：给 60～120 秒一关一个明确句号，然后立刻能进下一关。

```
┌─ 关卡重组完成 ─────────────────────────┐
│  用时  01:04                            │
│  死亡    2                              │
│  金币  +38                              │
│  评价  ★★★☆  （按时间/死亡简单折算）     │
│                                        │
│  ▶ 下一关                               │
│    再玩本关                             │
│    返回标题                             │
└────────────────────────────────────────┘
```

- 入场：青色 Arcana 符文碎裂 → 琥珀「完成」字标落下。  
- 数字滚动用等宽字阶跃增加（非平滑 lerp）。  
- 三星评价用简单阈值即可，避免复杂段位系统抢节奏。  
- 主 CTA 永远是「下一关」。

---

### 3.11 游戏结束 / 本跑总结（Run Summary）

当生命耗尽且不允许续关，或玩家主动结束本跑时：

- 展示本跑种子、总关数、总时长、总金币、最远章节。  
- 提供「用同一种子再开」「换种子」「回标题」。  
- 视觉比单关结算更「史诗」一点：铁头坐在重组废墟剪影前，但仍保持单面板，不加多卡墙。

---

### 3.12 图鉴 / 收藏（可选后置）

**目的**：查看敌人、陷阱、盔甲部件、已遇提示。

- 左列表右详情；详情区大图 + 3～5 行说明。  
- 未解锁条目为剪影 +「？？？」。  
- 风格与设置页同一套面板组件，保证全家桶一致。

---

### 3.13 制作人员（Credits）

- 自下向上像素卷动，或分页静态。  
- 背景用标题页远景的降饱和版本。  
- 末页回到标题；支持 Esc 退出。

---

### 3.14 退出确认 / 通用模态

所有破坏性操作（退出游戏、放弃本跑、恢复默认键位）统一：

```
┌─────────────────────┐
│  确定要离开吗？        │
│  [取消]   [确定]      │
└─────────────────────┘
```

- 默认焦点在「取消」。  
- 快捷：Esc = 取消；确认键在「确定」上才真正执行。

## 4. 通用 UI 组件库

实现时抽成可复用场景 / Control 主题，避免每页重做。

| 组件 | 规格 |
|------|------|
| `PanelFrame` | 9-slice 像素框，内边距 8px |
| `MenuList` | 文字项 + 左箭头焦点；支持禁用态（muted + 无箭头） |
| `PixelButton` | 仅用于单主 CTA；高度 16/24；琥珀填充 / 钢蓝描边次按钮 |
| `TabBar` | 底边 1px 下划线指示 |
| `SliderBar` | 轨道 4px 高，块状填充 |
| `Toggle` | 12×12 勾选框 |
| `DialogBox` | 底栏 48px，头像槽 32×32 |
| `HeartGlyph` / `CoinGlyph` | HUD 图集 |
| `Toast` | 顶中短暂提示，最多一行，2s |

**焦点态统一**：琥珀；**危险操作**：危险红描边；**魔法/系统**：青色。

## 5. 场景与角色视觉（局内非 UI）

### 5.1 角色「铁头」

- 剪影可读：圆盔 + 宽肩甲，远距离仍能辨认。  
- 状态帧：待机、跑、跳、落、蹬墙/二段、受伤、死亡、踩踏成功。  
- 面甲开合：受击闭合、过关微张、对话时随情绪切换。  
- 受击闪白 2 帧；无敌帧半透明闪烁（棋盘透明度，非平滑 fade）。

### 5.2 敌人（MVP 三类）

| 类型 | 视觉关键词 | 危险提示 |
|------|------------|----------|
| 地面巡逻 | 低矮甲虫/小卒，左右踱步 | 触碰红色轮廓 |
| 飞行 | 小型构装鸟/符文球 | 轨迹预示用淡青残影 |
| 远程 | 固定炮台/法阵 | 弹幕有 4 帧预警闪烁 |

被踩时统一「压扁 2 帧 → 爆成像素屑」。

### 5.3 关卡可读性

- 可站立平台：顶缘高光 1px，与背景明度差 ≥ 30%。  
- 致命区：尖刺、锯齿、虚空用 `--danger` 或高对比轮廓。  
- 移动平台：边缘琥珀微光，移动时拖尾不超过 2px。  
- 终点：旗帜 / 传送门，青色呼吸光，入场即能在构图右侧或高处找到。

### 5.4 背景分层

1. 远景剪影（城堡/村庄）慢视差  
2. 中景建筑构件  
3. 近景轻雾或飘尘（粒子极少）  
章节只换色与远景素材，平台碰撞语言保持一致。

## 7. 响应式与多分辨率

- 只做 **整数倍** 放大；窗口非整数倍时黑边 letterbox，不拉伸变形。  
- UI 全部在 320×180 逻辑坐标排布，再用 SubViewport / 拉伸模式输出。  
- 键鼠与手柄共用同一焦点系统；移动端若后置，虚拟键覆盖层不得侵入中央跳跃区。

---

## 8. 无障碍与可读性

- 色盲模式：危险物叠加斜线图案，不单靠颜色。  
- 可关闭闪白与震动。  
- HUD 放大选项：生命与金币字号 +2px。  
- 对话支持立刻显示整句。
